# Design Modernization Audit

이 문서는 MARU의 최종 미술 방향을 정하지 않는다. 현재 렌더링과 소스에서 확인되는 구조적 디자인 문제와 검증 기준만 기록한다. 브랜드·사용자 컨텍스트는 `.impeccable.md`가 확정된 뒤 이 문서에 연결한다.

## 현재 근거

- 프로덕션 실행 Chromium 캡처: `output/playwright/final-home.png`
- 상품·장바구니·체크아웃 캡처: `output/playwright/03-product.png`, `04-cart.png`, `05-checkout.png`
- 관리자 캡처: `output/playwright/12-admin-dashboard.png`, `13-admin-member-statistics.png`
- 활성 디자인 기준: `docs/design-system.md`

## 정량적 패턴

`apps/storefront/src` 기준:

- `rounded-[24px]`: 77회
- `rounded-[28px]`: 45회
- `rounded-[32px]`: 12회
- `rounded-[36px]`: 34회
- `surface-card`: 44회
- `admin-card`: 21회

대부분의 화면에서 콘텐츠의 중요도와 무관하게 동일한 대형 라운드·테두리·표면 처리가 반복된다. 이 때문에 페이지 간 구분은 생기지만 페이지 내부의 정보 위계는 약해진다.

## 스토어프런트 관찰

### 전역 셸

- 데스크톱 헤더는 브랜드, 카테고리, 계정, 장바구니만 제공하며 전역 검색 진입이 없다.
- 모바일에서는 카테고리 탐색이 사라지고 계정·장바구니·로그인이 동일 수준으로 남는다.
- 브랜드 워드마크와 `Digital Atelier`가 작아 첫 화면에서 브랜드 식별력이 약하다.

### 홈

- 히어로 아래에 카테고리, 추천, 또 다른 추천, 기획전, 신상품, 인기상품이 연속해 동일한 상품을 반복한다.
- 첫 화면의 주 행동은 검색과 히어로 CTA로 분산된다.
- 제품 사진은 충분하지만 섹션별 자르기·크기·텍스트 위치가 비슷해 이미지가 내러티브보다 카드 배경으로 소비된다.
- 운영 데이터에서 생성된 제목이 그대로 노출될 수 있어 브랜드 카피 품질이 일정하지 않다.

### 상품 목록과 카드

- 모든 카드가 이미지, 배지, 찜, 설명, 가격, 장바구니 버튼을 동일 밀도로 노출한다.
- 배지와 찜이 이미지 상단에서 경쟁하며, 장바구니 버튼 반복으로 목록의 시각적 소음이 커진다.
- 상품명·가격·리뷰·재고의 우선순위가 화면마다 일관되지 않다.

### 상품 상세

- 핵심 정보와 상세 설명이 각각 큰 카드에 들어가 이중 컨테이너처럼 보인다.
- 이미지 갤러리보다 하나의 고정 이미지가 큰 면적을 차지한다.
- 구매 행동, 재고, 배송·반품 신뢰 정보가 하나의 명확한 구매 패널로 정리되지 않는다.

### 장바구니·체크아웃·계정

- 장바구니의 주문 요약은 유효하지만 추천 그리드가 결제 작업보다 더 큰 시각 면적을 차지한다.
- 체크아웃은 단계별 정보가 각각 큰 카드에 들어가 한 화면에서 흐름이 끊긴다.
- 계정 화면은 중첩 카드와 24–36px 라운드가 반복되어 실제 정보량보다 화면이 길다.

## 관리자 관찰

- 모든 관리자 경로 상단에 큰 어두운 소개 영역과 pill 메뉴가 반복된다.
- 대시보드는 실제 KPI보다 소개 제목이 가장 큰 시각 요소다.
- KPI, 메모, 최근 주문이 모두 큰 카드라 비교·스캔보다 개별 콘텐츠 소개처럼 보인다.
- 상품·회원·리뷰 화면은 데스크톱 테이블 역할의 행도 각각 둥근 카드로 표시한다.
- 운영 화면의 설명 문장이 길어 필터·상태·주요 액션보다 먼저 읽힌다.
- 모바일 관리 화면은 대형 히어로와 가로폭 고정 테이블을 함께 유지해 작업 효율이 낮다.

## 디자인 구현 순서

1. `.impeccable.md`에 사용자·브랜드·미술 방향 확정
2. 두 서체, OKLCH 색상, 4pt 공간, radius, 표면, 모션 토큰 재정의
3. 전역 헤더·모바일 탐색·전역 검색·푸터 개편
4. 홈을 히어로 / 카테고리 / 하나의 상품 증거 / 하나의 에디토리얼 / 최종 CTA로 축소
5. 상품 카드와 목록의 정보 위계·상호작용 통일
6. 상품 상세를 미디어 영역과 구매 패널 중심으로 재구성
7. 장바구니·체크아웃·계정의 중첩 카드 제거와 작업 흐름 압축
8. 관리자 히어로를 제거하고 사이드바·상단바·작업 영역 구조로 전환
9. Chromium·WebKit desktop/mobile 시각 회귀 검증

## 완료 기준

- 390px, 768px, 1280px, 1440px에서 핵심 기능이 숨지 않고 가로 스크롤이 없다.
- 홈 첫 viewport에서 브랜드, 핵심 제안, 대표 이미지, 주 CTA가 동시에 식별된다.
- 홈의 동일 상품 반복 노출과 목적이 겹치는 섹션이 제거된다.
- 일반 콘텐츠에 24px 이상 radius를 사용하지 않고, 큰 프로모션 표면에만 제한한다.
- 관리자 각 화면은 첫 viewport에서 페이지 제목, 주요 액션, 필터 또는 KPI를 보여준다.
- 관리자 데이터 행은 카드가 아니라 비교 가능한 행·테이블·목록 구조를 사용한다.
- 모든 텍스트·상태·포커스가 WCAG AA를 충족한다.
- `prefers-reduced-motion`에서 정보 손실 없이 동작한다.
- 기존 Playwright 기능 스위트가 통과하고, 주요 화면의 전후 캡처를 남긴다.

## 구현 및 QA 결과

2026-07-11 기준으로 다음 변경을 적용했다.

- `.impeccable.md`에 고객, 구매 상황, 브랜드 성격, 시각 논제, 콘텐츠·상호작용 원칙을 확정했다.
- 스토어프런트 토큰을 warm ivory / softened ink / oxblood 기반 OKLCH 체계로 전환했다.
- Gowun Batang과 Noto Sans KR 조합으로 편집형 제목과 거래 UI를 분리했다.
- 헤더·푸터·홈·검색·카테고리·상품 상세·장바구니·체크아웃·계정·주문·인증 화면을 새 시스템으로 정리했다.
- 관리자 대형 히어로와 pill 내비게이션을 제거하고, 240px 워크스페이스 내비게이션·KPI·큐·테이블 구조를 적용했다.
- 모바일 검색 아이콘에 접근 가능한 이름을 추가하고 390px Chromium/WebKit에서 가로 오버플로가 없음을 확인했다.
- 테스트 실행 중 남은 임시 배너·카테고리·상품 카피를 V26·V27 마이그레이션으로 정리하고, 카테고리 E2E가 기존 데이터를 변경하지 않도록 생성·삭제 흐름을 격리했다.
