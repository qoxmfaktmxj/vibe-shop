# Production Readiness Review

외부 점검 제안을 현재 코드와 대조한 결과와 적용 범위를 기록한다.

## 적용 완료

- 비회원 주문 목록의 전화번호 단독 조회 제거
- 주문별 20분 HttpOnly 접근 쿠키, 실패 제한, 접근 감사 로그 도입
- 비회원 주문 상세 개인정보 마스킹
- 체크아웃 개인정보의 `localStorage` 저장 제거 및 구매 동의/유효성 차단
- 조건부 원자 재고 차감, 취소 행 잠금, 재고 1회 복원
- 병렬 idempotency 충돌 시 기존 주문 반환 및 UUIDv7 주문번호 적용
- production 모드의 mock 결제 활성화 차단과 데모 전역 안내
- 가격 preview와 추천 요청 분리, preview 실패 시 체크아웃 차단
- 장바구니 mutation 직렬화, 오류 노출, 계정 전환 초기화
- 품절 상품 구매 차단과 추천 실패의 상품 상세 격리
- 관리자 상품 상세 설명의 조회/수정 계약 일치 및 강조색 검증
- 색상 대비, skip link, 폼 레이블, live/alert 상태 보강
- 운영 CORS allowlist가 환경값을 그대로 따르도록 수정하고 기본 보안 헤더 추가
- Next.js 16.2.10, Spring Boot 4.0.7로 유지보수 버전 갱신
- Chromium 전체 흐름, WebKit 모바일, production-start E2E 구성

## 판단상 부분 적용

- 인증/관리자 보안 묶음은 하나의 P0로 보기보다 항목별 위험이 다르다. 이번 범위에서는 CORS, 보안 헤더, 쿠키 속성, 의존성 취약점과 비회원 주문 접근 제어를 반영했다.
- mock 결제는 데모 환경에서는 의도된 기능이다. 실서비스 위험은 production에서 활성화될 때이므로 런타임 시작 차단으로 경계를 명확히 했다.
- 기존 idempotency unique 제약은 중복 행을 막고 있었다. 실제 결함은 병렬 충돌이 5xx로 노출되는 점이어서 기존 주문 반환으로 수정했다.

## 별도 의사결정이 필요한 후속 범위

- 관리자 MFA, 세션 절대/비활성 만료, 중요 작업 재인증, 세부 RBAC, 관리자 감사 로그
- 관리형 이미지 업로드 공급자와 파생 이미지 정책
- 관리자 상품/리뷰 목록 서버 페이지네이션
- 모바일 전역 메뉴와 검색 정보 구조 개편
- 상품 옵션/SKU/다중 이미지/배송 모델 확장
- 검색 자동완성, 가격·평점·소재 필터와 최근/인기 검색어

위 항목은 유효한 제품·운영 제안이지만, 인증 공급자·스토리지·상품 모델·탐색 정책 선택 없이 구현하면 추측성 구조가 된다. 별도 요구사항과 수용 기준을 확정한 뒤 진행한다.

## 검증 기준

- `npm run qa`: 통과
- root 및 `apps/storefront`의 `npm audit`: 취약점 0건
- PostgreSQL 재고 10개에 50개 병렬 주문: 성공 10, 재고 부족 40, 최종 재고 0
- 동일 idempotency key 100개 병렬 주문: 응답 100개가 주문 1건, 결제 1건, 재고 1회 차감으로 수렴
- 동시 취소 2개: 성공 1, 거절 1, 재고 1회 복원
- production + mock 결제 조합: API 시작 차단 확인
- Playwright production-start: Chromium 12건, WebKit 모바일 2건, 합계 14건 통과
