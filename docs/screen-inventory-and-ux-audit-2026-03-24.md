## 3. 전반적인 UX 진단 요약

현재 Vibe Shop은 기능적으로는 꽤 많이 구현되어 있지만, UX 관점에서는 다음 특징이 강하다.

### 좋은 점
- 검색/카테고리/상품상세/장바구니/주문/조회 흐름이 한 바퀴 돈다.
- 추천/최근 본 상품/찜/리뷰/계정/주문조회가 연결되어 있어 **데모 관점의 완성도**가 있다.
- 관리자도 단순 CRUD를 넘어서 **운영 보조 대시보드**까지 들어와 있다.

### 현재 강한 문제점
1. **홈에서 사용자의 첫 행동이 분산됨**
   - 컬렉션 보기
   - 검색으로 찾기
   - 헤더 Search
   - 카테고리 직접 선택
   - 추천 선반 클릭
   사용자 입장에서 출발점이 너무 많다.

2. **Search 진입이 중복됨**
   - 헤더 네비에 Search
   - 우측 액션에도 Search
   - 홈 히어로에도 Search
   → 정보 구조상 중복이다.

3. **컬렉션 탐색 vs 검색 탐색의 위계가 불명확함**
   - 지금은 둘 다 강하게 밀고 있는데,
   - 실제 사용자 친화 UX로 가려면 **대표 진입축을 하나 정하고** 나머지는 보조로 내려야 한다.

4. **용어가 아직 운영자/개발자 관점인 부분이 있음**
   - `Lookup`
   - `Journal`
   - `Product Match`
   - `Continue Exploring`
   - 일부 Search filter chip 값
   사용자 친화적 라벨링으로 재정리 여지가 큼.

5. **페이지 간 시각 언어는 통일되어 있지만, 목적별 구분은 약함**
   - 거의 모든 화면이 같은 카드/에디토리얼 톤을 써서 예쁘긴 한데,
   - 검색 화면 / 주문 화면 / 계정 화면 / 운영 화면의 목적 차이가 UI 구조에서 덜 드러난다.

## 4. 석이 말한 UX 개선 방향에 대한 즉시 판단

질문 요지:
1. 검색 관련 중복 버튼이 있다.
2. 홈에서 꼭 Search 페이지를 거쳐야 하는 현재 구조를 더 친숙하게 바꾸고 싶다.
3. Search를 더 직접적이고 쉬운 행동으로 만들고 싶다.

### 내 판단
이 방향이 맞다.

특히 아래 3개는 우선순위가 높다.

#### A. 홈을 “검색 시작 화면”으로 승격
현재 홈은 에디토리얼 랜딩 성격이 강하다. 하지만 실제로는 상품 탐색형 서비스라서,
**홈 Hero에서 바로 검색 입력이 가능한 구조**가 더 직관적이다.

권장:
- Hero CTA 버튼 2개 대신
  - **메인 검색 입력창 1개**
  - 추천 검색어/빠른 필터 칩
  - 보조 CTA로 카테고리 보기
- 즉 `SEARCH`를 누르면 `/search`로 가는 것뿐만 아니라,
  **홈에서 바로 키워드 입력 → `/search?q=...` 이동**이 되게 하는 것이 좋다.

#### B. Search를 “기능 페이지”가 아니라 “기본 탐색 모드”로 재정의
지금은 Search가 별도 페이지처럼 분리되어 있는데,
향후 UX에서는 아래 중 하나를 선택하는 게 좋다.

1. **Search-first 상점**
   - 홈 상단부터 검색 중심
   - 카테고리는 서브
2. **Collection-first 상점**
   - 홈은 큐레이션 중심
   - 검색은 도구성 기능

현재 구현은 둘을 동시에 강하게 밀고 있어서 다소 어정쩡하다.

#### C. 중복 Search 진입 제거
현재는 Search가 너무 여러 군데 있다.

권장:
- 헤더 메인 네비의 Search 1개 유지
- 우측 별도 Search 버튼 제거
- 홈 히어로의 `검색으로 찾기`는 버튼이 아니라 **입력형 search box**로 승격

즉,
- **Search 링크는 1개**
- **Search 행동은 어디서든 직관적**
이 상태가 좋다.

## 5.1 글로벌 헤더 / 글로벌 탐색 구조

파일:
- `apps/storefront/src/components/shell/site-header.tsx`

### 현재 역할
모든 storefront 화면에서 공통으로 쓰이는 상단 탐색 허브.

### 현재 주요 요소
#### 상단 좌측
- 브랜드명: `Vibe Shop`
- 서브카피: `Digital Atelier`
- 홈 링크 역할

#### 상단 중앙 메인 내비
- Home
- Search
- Orders
- Lookup
- Journal

#### 상단 우측 액션
- 로그인/회원가입 또는 계정/로그아웃
- 별도 Search 버튼
- Guest 버튼
- Bag 버튼

#### 하단 카테고리 네비
- 리빙
- 키친
- 웰니스

### 제품적 역할
- 브랜드 인지
- 전역 이동
- 검색 진입
- 비회원 주문조회 진입
- 장바구니 진입
- 카테고리 탐색

### UX 문제
1. **Search가 2중 배치됨**
   - 중앙 메뉴 Search
   - 우측 별도 Search
2. **Lookup / Guest가 분리되어 의미 중복처럼 보임**
   - `Lookup`과 `Guest`는 둘 다 비회원 주문조회 계열로 해석될 여지가 큼
3. **Journal은 실제로 FAQ인데 라벨이 다름**
   - 사용자는 콘텐츠/블로그를 기대할 수 있음
4. **탐색 축이 너무 많다**
   - Home / Search / Orders / Lookup / Journal / category / Guest / Bag

### 디자인 개편 포인트
- 헤더는 **역할을 3개로 축소**하는 것이 좋음
  1. 탐색
  2. 검색
  3. 계정/장바구니
- `Lookup` → `비회원 주문조회` 또는 `주문조회` 식으로 명확화
- `Journal`은 실제 성격에 따라 `FAQ` 또는 `도움말`로 바꾸는 것이 자연스러움
- Search를 링크가 아니라 **검색 아이콘 + 확장 검색창**으로 재설계 가능

## 5.2 홈 `/`

파일:
- `apps/storefront/src/app/page.tsx`

### 현재 역할
브랜드 인트로 + 큐레이션 랜딩 + 추천 진입점.

### 현재 주요 블록
1. **Hero 영역**
   - 시즌/에디토리얼 카피
   - 큰 타이틀
   - 설명 문구
   - CTA 2개
     - primary: 운영 설정값 기반 CTA (`home.heroCtaHref`)
     - secondary: `/search` 로 가는 `검색으로 찾기`
   - 우측 히어로 비주얼/배너

2. **Hero 하이라이트 카드**
   - 프로모션성 배너/서브 강조

3. **카테고리 셀렉션**
   - 대표 카테고리 카드 3개

4. **최근 본 상품 선반**
   - recently viewed

5. **추천 선반**
   - 최근 본 상품이 있으면 continue exploring
   - 없으면 trending for you

6. **프로모션 섹션**
   - campaign형 카드

7. **큐레이션 상품열 / 신상품 / 베스트셀러**
   - 하단 상품 카드 영역들

### 제품적 역할
- 첫 인상 형성
- 현재 운영자가 밀고 싶은 배너 노출
- 카테고리/검색/추천/프로모션으로 분기

### 현재 장점
- 데모 화면으로는 인상적이고 풍부함
- 추천/최근 본 상품을 홈에서 잘 드러냄
- 브랜드 톤이 분명함

### 현재 UX 문제
1. **행동 우선순위가 불명확함**
   - 컬렉션 보기
   - 검색으로 찾기
   - 카테고리 셀렉션
   - 추천 선반
   - 프로모션
   동시에 많다.

2. **Hero CTA가 제품 탐색 방식과 안 맞을 수 있음**
   - primary CTA가 CMS형/배너형이고
   - secondary CTA가 search라서
   - 사용자 입장에서 “어디서부터 시작하지?”가 남음

3. **검색은 중요 기능인데 화면상 주인공이 아님**
   - 지금은 링크 CTA 수준
   - 실제 상품 쇼핑 UX에서는 검색이 더 전면에 나와도 됨

### 추천 개편 방향
#### 방향 1: Search-first Home
- Hero에 큰 검색창 배치
- placeholder 예시 유지: `여름 리빙 10만원 이하 베이지 선물`
- 검색창 아래 빠른 칩 배치
  - 여름
  - 리빙
  - 선물
  - 10만원 이하
  - 베이지
- 카테고리 카드는 아래로 내림
- 기존 `검색으로 찾기` 버튼 제거

#### 방향 2: Collection-first Home
- Hero는 큐레이션/프로모션 전용
- Search는 헤더 전역 검색으로 이동
- 홈 CTA는 모두 컬렉션/카테고리 중심

### 내 추천
현재 Vibe Shop은 search/recommendation 구현이 꽤 괜찮아서,
**Search-first Home**으로 재설계하는 편이 더 사용자 친화적이다.

## 5.3 검색 `/search`

파일:
- `apps/storefront/src/app/search/page.tsx`
- `apps/storefront/src/components/search/search-form.tsx`

### 현재 역할
텍스트 검색 + 자연어 검색 + 카테고리 필터 + 결과 목록 허브.

### 현재 주요 블록
1. 상단 소개 카드
   - 제목: 상품 검색
   - 설명: 상품명/카테고리/분위기 키워드 검색

2. SearchForm
   - keyword input
   - category select
   - submit button
   - 자연어 검색 설명 텍스트

3. 검색 결과 영역
   - 결과 제목
   - 선택 카테고리 표시
   - 상품 개수
   - 적용된 필터 chip
   - fallback 안내 배너
   - sort tabs
   - reset link
   - 결과 상품 grid

4. 검색 전 상태
   - 카테고리 카드 리스트

### 제품적 역할
- 탐색형 사용자의 핵심 진입점
- 자연어 검색 데모 포인트
- 필터 해석 결과를 사용자에게 피드백하는 설명형 화면

### 현재 장점
- 기능적으로 좋다.
- 단순 키워드 검색이 아니라 **해석된 검색**이 드러난다.
- fallback 설명이 있어 모델 없는 rule-based 검색도 꽤 똑똑해 보인다.

### 현재 UX 문제
1. **검색 전 빈 화면이 카테고리 화면과 역할이 겹침**
   - `/search`에 들어가도 아직 검색을 안 했으면 카테고리 카드가 나온다.
   - 그런데 `/category/[slug]`도 이미 존재한다.
   - 즉 검색 화면이 “카테고리 입구” 역할까지 같이 하고 있다.

2. **검색/컬렉션 경계가 모호함**
   - keyword search
   - category browse
   - sort
   - filter chip
   이 모두 한 화면에 있어 정보량이 많다.

3. **필터 chip 라벨의 사용자 친화도가 아직 낮을 수 있음**
   - 예: `beige`, `summer`, `living` 같은 값은 내부 모델 느낌이 남는다.

### 추천 개편 방향
- Search 화면을 명확히 **탐색 조작 화면**으로 정의
- `/search` 진입 즉시 포커스되는 검색창 제공
- 검색 전 상태는 다음 둘 중 하나로 단순화
  1. 최근 검색 / 추천 검색어
  2. 빠른 필터 칩
- 카테고리 카드는 검색 화면의 기본 상태에서 제거해도 됨
- category browse는 `/category/[slug]`에 더 맡기는 편이 구조상 깔끔함

### 강한 제안
- 홈 Hero에서 바로 검색하면 `/search?q=...`로 이동
- `/search`는 곧바로 “결과를 다루는 작업공간”처럼 느껴지게 개편
- 즉 홈은 **입력 시작**, 검색은 **정제/비교/결정** 역할 분리

## 7. 디자인 전면 개편 시 반드시 전달할 핵심 요구사항

디자인 전문 AI에게 넘길 때 아래 요구를 함께 주는 것이 좋다.

### 7.1 제품 방향성
- Vibe Shop은 단순한 예쁜 랜딩이 아니라
  **검색 + 카테고리 탐색 + 추천 + 장바구니 + 주문 + 계정 + 운영콘솔**이 있는 커머스 서비스다.
- 따라서 단일 Hero 중심 랜딩이 아니라,
  **탐색성과 전환성**을 동시에 높이는 구조가 필요하다.

### 7.2 반드시 해결해야 할 UX 과제
1. Search 진입 중복 제거
2. Home에서 직접 Search 시작 가능하게 만들기
3. 카테고리 탐색과 검색 탐색의 역할 분리
4. Lookup/Guest/Journal 같은 라벨 정리
5. 계정 화면 정보 밀도 재구성
6. Admin은 storefront와 다른 운영 도구 톤 강화

### 7.3 추천하는 상위 UX 컨셉
#### 안 1. Search-first Commerce
- 홈 Hero = 큰 검색창
- 카테고리/추천은 검색 아래 보조 탐색
- 사용자 행동 중심 구조

#### 안 2. Editorial Commerce
- 홈 Hero = 큐레이션 중심
- Search는 전역 헤더 중심
- 카테고리/배너/프로모션 중심 탐색

### 내 추천
현재 구현과 사용자 요청을 합치면,
**Search-first Commerce**가 더 적합하다.

## 10. 바로 실행 가능한 UX 개편 제안 요약

### 제안 A. 홈 Hero를 검색 시작점으로 교체
- 버튼 2개 → 검색창 1개 + 보조 CTA 1개
- 예시 placeholder 유지
- 검색어 입력 즉시 `/search?q=...`

### 제안 B. 헤더 단순화
- Search는 한 곳만 유지
- Lookup/Guest 통합
- Journal → FAQ/도움말로 명확화

### 제안 C. Search를 결과 작업공간으로 강화
- 검색 전 상태의 카테고리 카드 축소
- 최근 검색 / 추천 키워드 / 빠른 필터 중심으로 재구성

### 제안 D. 계정 화면 탭화
- 프로필 / 배송지 / 주문 / 찜 / 리뷰

### 제안 E. Admin은 운영툴 정체성 강화
- storefront처럼 예쁜 카드보다는
- 더 빠른 탐색/필터/액션 중심 구조 검토

## 11. 결론

현재 Vibe Shop은 기능적으로는 이미 상당히 많이 구현되어 있다.
하지만 UX는 아직
- 에디토리얼 랜딩
- 탐색형 커머스
- 운영형 콘솔
이 세 가지 성격이 동시에 섞여 있다.

따라서 디자인 전면 개편의 핵심은 **예쁘게 만드는 것**이 아니라,

1. 사용자의 첫 행동을 더 명확하게 만들고
2. 검색/컬렉션/추천의 역할을 다시 나누고
3. 운영 도구와 고객용 화면의 정체성을 분리하는 것

에 있다.

이 문서는 그 재설계를 위한 **현행 구조 기준서**로 사용하면 된다.
